# YouEyeSea — Agent Context

## Project
Next.js 16.2 (App Router) + TypeScript + Tailwind v4.
UIC grade distribution viewer — search courses, view charts, leave comments/reviews.
Deployed on Vercel, Postgres via Prisma.

## Key Directories
- `app/_util/` — server-safe utilities: `formatGradeData` (Prisma aggregate → chart data), `gpaCalculator` (weighted GPA), `postComment`/`postReview` (server actions), `semesterToNumber` (sort key), `getProfessorNameFromID`
- `components/ui/` — shadcn/ui radix-based components
- `components/custom/` — feature components grouped by pane/feature
- `components/custom/skeleton/` — reusable loading-state components extracted from inline shimmers, used in `loading.tsx` and Suspense fallbacks
- `lib/` — `prisma.ts` (client + `prismaCacheStrategy`), `utils.ts` (`cn`)
- `prisma/` — schema, migrations, seed scripts, sample CSVs
- `hooks/` — `use-mobile.tsx` (`useIsMobile()`)
- `scripts/` — `generate-cache.ts`
- `cache/` — production + sample cache arrays (committed), barrel files merge them in dev

## Data Flow
Server Components query Prisma → pass data as props to Client Components.
No API routes (except `/api/og` for page-specific OpenGraph images). URL search params drive all filters.

## Prisma
- v7: needs adapter for local development (`@prisma/adapter-pg`) or Accelerate in production (`@prisma/extension-accelerate`)
- `DATABASE_URL` must be `postgres://` (dev) or `prisma+postgres://` (prod)
- Use `...prismaCacheStrategy(ttl, swr)` for Accelerate caching (empty object in dev)
- Local db: `npx prisma dev --name="youeyesea"` → copy `postgres://` URL to `.env`

## Caching
Two cache layers for grade data:

- **`'use cache'`** + `cacheLife('semesterly')` on `getCourseDetails`, `getCourseInstance`, `getGradeTotalsForSubject` — Next.js Data Cache. Profile: stale 30d / revalidate 90d / expire 180d. Tagged `'grade-data'` for `revalidateTag()` on data uploads.
- **`prismaCacheStrategy(604800, 86400)`** on every Prisma read query — Accelerate edge cache as second layer.

Comments and reviews skip both layers — they use user-submitted content that should appear immediately.

## Component Pattern
- **Server Components**: fetch data from Prisma, pass results as props
- **Client Components**: all UI rendering and interactivity (charts, forms, tables, dropdowns)
- Each data pane: `*Server.tsx` (async, Prisma) → `*Pane.tsx` (client, receives props)
- **Sync Server Components**: not all server components need `async`. `CourseHeaderTitle` reads the course title from the static `@/cache/course-list` synchronously — renders instantly, no DB round-trip.
- **Split Suspense**: title data available from static cache renders outside Suspense; controls that need DB (dropdowns, actions) suspend independently.

## Conventions
- `"use client"` only when React hooks or browser APIs are needed
- Named exports: `function Foo() {}` then `export { Foo }` at bottom
- `@/` path alias throughout
- Tailwind v4 with CSS variables for UIC colors (`--color-uic-red-*`, `--color-uic-navy-*`)
- Recharts for charts, `motion/react` for animation, MiniSearch for fuzzy search

## Notable
- Recharts SSR warning silenced via `patches/recharts+3.8.1.patch` (auto-applied by `postinstall`)
- `getCourseDetails` in `app/_util/getCourseDetails.ts` uses `React.cache()` + `'use cache'`
- `process.env.NODE_ENV === "development"` used throughout to switch behavior: Prisma adapter vs Accelerate, `prismaCacheStrategy` no-op, sample cache merging (handled by `cache/*.ts` exporter files), trending courses fallback — always check for this pattern before adding dependencies
- Semesters are stored as `"fall_2025"`, `"spring_2025"` etc. — `semesterToNumber()` converts to numeric sort keys (`20253`), `cleanSemesterName()` handles display (`"Fall 2025"`), optionally `{ format: "mini" }` for `"Fall '25"`

## Environment Variables

- `DATABASE_URL` — `postgres://` for local dev, `prisma+postgres://` for Vercel/Accelerate
- Local: `npx prisma dev --name="youeyesea"` → copy the `postgres://` URL into `.env`
- See `.env.example` for the template

## Things to Avoid

- `components/ui/` — shadcn components, use as-is or regenerate via CLI, avoid manual edits
- `src/generated/prisma/` — auto-generated by Prisma, never edit manually
- `cache/production/course-list.ts` etc. — auto-generated caches, regenerate after seeds (if-needed) via `npm run generate-cache`
- `cache/sample/` — sample caches, should not need to be regenerated
- `patches/recharts+3.8.1.patch` — used to suppress a common recharts warning, do not remove
- `prisma/grade_distribution_data/` — production CSVs, use `sample_grade_distribution_data/` for local development

## Coding Conventions

- **Components:** PascalCase (`GradeDistributionChart`, `TablePaneServer`), file matches component name
- **Filenames:** kebab-case (`table-pane-server.tsx`, `grade-distribution-chart.tsx`)
- **Utilities:** camelCase (`formatGradeData`, `calculateGPA`, `semesterToNumber`)
- **Exports:** Named exports at bottom (`export { GradeDistributionChart }`), no default exports for components
- **Server/Client split:** `*Server.tsx` fetches data → `*Pane.tsx` renders it (client) — e.g. `table-pane-server.tsx` → `table-pane.tsx`
- **`components/custom/`** — app-specific feature components, grouped by pane
- **`components/ui/`** — shadcn primitives, use as-is
- **`app/_util/`** — server-safe utility functions only
- **Interfaces/Types:** PascalCase, co-located with usage
- **`"use server"`** — only in `app/_util/` for server actions (`postComment`, `postReview`)
- **URL search params** — drive all filter state, avoid useState for filter values to ensure links are shareable
- **let / const** — prefer `const`, avoid `let` for non-mutating values
